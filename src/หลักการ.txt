Model = เกี่ยวกับฐานข้อมูล: ใช่แล้ว! Model ทำหน้าที่จัดการข้อมูลและตรรกะทางธุรกิจ (Business Logic) รวมถึงการติดต่อกับฐานข้อมูลหรือการคำนวณข้อมูลต่าง ๆ
View = หน้า page: ใช่เช่นกัน! View คือส่วนที่ผู้ใช้เห็นและโต้ตอบ เช่น หน้าเว็บที่แสดงข้อมูลหรือฟอร์มต่างๆ
Controller = เปรียบเสมือนการจัดการฟังก์ชันที่เกี่ยวข้องกับการเชื่อมต่อระหว่าง View และ Model ซึ่งทำให้โค้ดมีการจัดการที่เป็นระบบและสามารถใช้ซ้ำได้ง่าย

สรุป

Model = DB
View = PAGE 
Controller = function,class,oop
